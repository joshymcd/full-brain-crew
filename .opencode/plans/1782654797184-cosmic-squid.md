# Plan: Dummy Chat Route With shadcn AI Components

## Goal

Add a new dummy chat page at `/chat/$chatId` for the webapp. The page should use the new shadcn LLM/chat UI primitives with static test data only, so the UI can be shaped before wiring it to OpenCode.

## Context

- Web app root: `webapp/`
- Framework: React 19, Vite+, TanStack Router file routes
- Routes live in `webapp/src/routes/`
- Existing route pattern: `createFileRoute("/")({ component: Home })`
- Dynamic TanStack Router params use `$param`, so `chat/{chatId}` maps to route path `/chat/$chatId`
- `webapp/src/routeTree.gen.ts` is generated and should not be manually edited
- shadcn config exists at `webapp/components.json`
- Existing UI components include `button`, `avatar`, `input`, `sidebar`, `tooltip`, etc.
- Missing requested AI/chat primitives: `attachment`, `bubble`, `marker`, `message`, `message-scroller`
- Project guidance says to use Vite+ commands such as `vp dlx`, `vp run`, `vp check`, and `vp test`

## Files To Modify

- `webapp/src/routes/chat/$chatId.tsx`
- `webapp/src/routeTree.gen.ts` via route generation only
- shadcn-generated files under `webapp/src/components/ui/`, expected:
  - `attachment.tsx`
  - `bubble.tsx`
  - `marker.tsx`
  - `message.tsx`
  - `message-scroller.tsx`
- Potential dependency/lockfile changes from shadcn install, if required by generated components:
  - `webapp/package.json`
  - `webapp/pnpm-lock.yaml`

## Implementation Steps

1. Install the missing shadcn AI chat primitives from `webapp/`:

   ```bash
   vp dlx shadcn@latest add attachment bubble marker message message-scroller
   ```

   If the bare component names are not resolvable by the installed shadcn CLI/registry, retry with the correct shadcn registry identifiers for those components while still using `vp dlx`.

2. Create `webapp/src/routes/chat/$chatId.tsx` using TanStack Router:

   ```tsx
   import { createFileRoute } from "@tanstack/react-router"

   export const Route = createFileRoute("/chat/$chatId")({
     component: ChatPage,
   })

   function ChatPage() {
     const { chatId } = Route.useParams()
     // render dummy chat UI
   }
   ```

3. Add local dummy data in the route file:

   - A few assistant/user messages
   - At least one status/marker row such as “Explored 4 files” or “Running dummy tool call”
   - At least one attachment row to exercise the `Attachment` component
   - No OpenCode SDK calls, network requests, or persisted state yet

4. Compose the page with the shadcn primitives:

   - Outer route container: `flex h-full min-h-0 flex-col`
   - Header inside the route content showing `Chat {chatId}` and a “Dummy data” label
   - `MessageScrollerProvider`, `MessageScroller`, `MessageScrollerViewport`, `MessageScrollerContent`, `MessageScrollerItem`, and `MessageScrollerButton`
   - `Message` rows aligned by role: assistant at start, user at end
   - `Bubble`/`BubbleContent` as the visible message surface
   - `Marker` rows for system/status entries
   - `Attachment`/`AttachmentGroup` inside relevant message content
   - Footer composer mock with disabled input/textarea and disabled send button, clearly indicating OpenCode wiring is coming later

5. Keep styling minimal and aligned with the existing layout:

   - Do not change `webapp/src/routes/__root.tsx` unless the new route reveals an unavoidable height/layout issue
   - Avoid introducing a separate feature directory for this dummy page unless the route becomes too large
   - Use existing `@/components/ui/*` imports and `@/lib/utils` conventions

6. Regenerate TanStack Router routes from `webapp/`:

   ```bash
   vp run generate-routes
   ```

## Verification

Run from `webapp/`:

```bash
vp check
vp test
```

Manual verification:

```bash
vp dev
```

Open `http://localhost:3000/chat/demo` and confirm:

- The page renders under the existing site header/sidebar layout
- The `demo` route param appears in the page header
- Dummy user and assistant messages render in a scrollable chat transcript
- Attachment and marker examples are visible
- Composer controls are disabled or non-functional placeholders
- No OpenCode connection is attempted
