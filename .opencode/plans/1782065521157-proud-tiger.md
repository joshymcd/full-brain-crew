# Plan: Move OpenCode Client Configuration to Root `.env`

## Goal

Load OpenCode connection settings from the repository root `.env` (one level above `webapp`) and use those values in both:

- The webapp OpenCode SDK client created in `webapp/src/main.tsx`.
- The Docker/OpenCode server startup command in `scripts/entrypoint.sh`.
- Local Docker Compose services for both the OpenCode server and the webapp.

## Requirements

- Add/use these env vars:
  - `VITE_OPENCODE_SERVER_URL` for the browser SDK base URL, e.g. `https://brain.joshmcd.xyz/`.
  - `OPENCODE_PORT` for the host-side Docker port mapping.
  - `WEBAPP_PORT` for the host-side webapp port mapping.
- Do not expose Basic Auth username/password through Vite env.
- Prompt for username/password in the webapp and attach them to SDK requests via the router-context OpenCode client.
- Keep OpenCode SDK client in TanStack Router context (`main.tsx`) and keep routes consuming it via context.

## Important Note

Because the OpenCode SDK client runs in the browser, any credentials used by it are still visible to that browser session. They are no longer baked into env/build output; they are entered in the UI and stored in session storage.

## Files To Modify

### `webapp/vite.config.ts`

- Keep Vite defaults. Browser-visible config uses standard `VITE_*` env vars from `webapp/.env` or Docker Compose process env.

### `webapp/src/main.tsx`

- Remove hardcoded OpenCode URL and auth string.
- Read only the server URL from `import.meta.env`:
  - `import.meta.env.VITE_OPENCODE_SERVER_URL`
- Move client creation into an auth-aware helper/provider that can rebuild the SDK client when UI-entered credentials change.
- Keep OpenCode SDK access in TanStack Router context.

### `scripts/entrypoint.sh`

- Keep Docker/container runtime simple and consistent: OpenCode always listens on container port `8080`.
- Keep existing `PORT="${PORT:-8080}"` fallback if Railway/platform runtime needs it.
- Do not pass `OPENCODE_PORT` into the OpenCode command inside the container.
- Keep startup as:
  - `opencode web --port "${PORT}" --hostname 0.0.0.0`
- For local Docker Compose, set/keep `PORT=8080` inside the container if needed and map host port separately.

### `docker-compose.yml`

- Keep container ports hardcoded and simple:
  - OpenCode container port: `8080`
  - Webapp container port: `3000`
- Use env vars only at the host-to-container boundary:
  - OpenCode mapping: `"${OPENCODE_PORT:-8080}:8080"`
  - Webapp mapping: `"${WEBAPP_PORT:-3000}:3000"`
- Keep `env_file: .env` as-is.
- Add a separate `webapp` service for local development:
  - Build/run from `./webapp`.
  - Use the same root `.env` via `env_file: .env`.
  - Expose `${WEBAPP_PORT:-3000}:3000`.
  - Run Vite dev server bound to all interfaces: `vp dev --host 0.0.0.0 --port 3000`.
  - Mount `./webapp:/app` for local dev iteration and use an anonymous volume for `/app/node_modules` if needed.
- Add `depends_on: [app]` for convenience, so `docker compose up webapp` starts the OpenCode server too.

Cleaner Vite Docker dev recommendation:

- Use a lightweight Dockerfile in `webapp/Dockerfile.dev` rather than a long inline Compose command.
- It should:
  - Use `node:22-trixie-slim`.
  - Enable Corepack.
  - Set `WORKDIR /app`.
  - Copy package manifests if building dependencies into an image, or rely on the bind mount plus named `node_modules` volume for local dev.
  - Start `vp dev --host 0.0.0.0 --port 3000`.
- Compose remains readable and stable; the container always serves port `3000`.

Recommended local Compose shape:

```yaml
services:
  app:
    build: .
    ports:
      - "${OPENCODE_PORT:-8080}:8080"
    env_file:
      - path: .env
        required: false
    volumes:
      - ./.vault:/vault

  webapp:
    working_dir: /app
    image: node:22-trixie-slim
    command: sh -lc "corepack enable && pnpm install && pnpm exec vp dev --host 0.0.0.0 --port 3000"
    ports:
      - "${WEBAPP_PORT:-3000}:3000"
    env_file:
      - path: .env
        required: false
    volumes:
      - ./webapp:/app
      - webapp_node_modules:/app/node_modules
    depends_on:
      - app

volumes:
  webapp_node_modules:
```

If `vp` is not available through `pnpm exec`, use the project script instead and update `webapp/package.json` dev script to bind to `0.0.0.0`, or install/run Vite+ through the documented Vite+ wrapper available in the repo.
